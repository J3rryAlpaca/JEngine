# JCode Scripting Language Specification v1.0
This document outlines the official instruction set architecture, execution rules, and memory lifecycle protocols for the custom interpreted runtime environment.

---

## 1. Core Language Rules
* **Sequential Execution**: Instructions are parsed and executed line-by-line from top to bottom.
* **Strict Allocation Gating**: All environment entities (players, object meshes) must be initialized and declared *before* any properties (coordinates, attributes) can be modified. Violating this sequence forces a strict runtime reference exception.
* **Coordinate Space**: Operates on a standard 40-stud flat grid surface bounded between (0, 0) and (40, 40).

---

## 2. Global Instruction Set Reference

### playermodel <filename.obj>
Allocates and caches the 3D target .obj mesh file to serve as the visual layout frame for the player character.
Example: playermodel sphere.obj

### movetype <type>
Sets the global keyboard hardware input mode mapping rule for moving the player character matrix. Supports string tokens 'wasd' or 'arrow'.
Example: movetype wasd

### setplayerpos <axis> <value>
Snaps the player character's structural starting origin vector along a specific axis to the target stud value.
Example:
setplayerpos x 20
setplayerpos y 20

### cammode <mode_integer>
Configures the projection matrix perspective alignment. 
* Mode 1: 1st-Person Perspective (locked right to player origin eye level).
* Mode 3: 3rd-Person Fixed Overhead Orbit Perspective.
Example: cammode 3

### bgmloop <filename.wav>
Hooks the native Windows multimedia audio buffer stream to stream and infinitely loop a target uncompressed .wav track file asynchronously on a background thread without interrupting game loop cycles.
Example: bgmloop narbacular_up.wav

### assignobj <reference_name> <filename.obj>
The primary object memory allocation flag. Instantiates a custom entity record inside the engine state map under a string reference label, tying it to a target geometric mesh. This must run before modifying its attributes.
Example: assignobj box cube.obj

### setobjpos <reference_name> <axis> <value>
Translates an existing allocated object's coordinates straight to a target position along the chosen axis grid layout path.
Example: 
setobjpos box x 20
setobjpos box y 10

### objpushable <reference_name> <boolean_integer>
Sets the collision resolution flag for an entity.
* 1: Pushable. Enables physics calculations to translate velocity from the player's bounding sphere into the block.
* 0: Solid Anchor. Acts as an immovable physical barrier.
Example: objpushable box 1

### waituntiltouch <reference_name_1> <reference_name_2>
A blocking yielding loop instruction. Pauses the program counter's execution flow and holds the thread right on this line until the physical boundary distance checks between the two objects fall below the proximity limit.
Example: waituntiltouch box button

### displaytext "<string_message>"
Pushes a text overlay modal string directly over the 3D canvas rendering window matrix.
Example: displaytext "You win!"

## 3. Complete Test Level Example (BASE.jcode)
The following complete production script creates a classic sokoban-style block-pushing switch puzzle environment:

playermodel sphere.obj
movetype wasd
setplayerpos x 20
setplayerpos y 20
cammode 3
bgmloop narbacular_up.wav

// 1. ALLOCATE ENTITIES IN MEMORY FIRST
assignobj box cube.obj
assignobj button button.obj

// 2. CONFIGURE SURFACE COORDINATES & PHYSICAL VALUES
setobjpos box x 20
setobjpos box y 10
setobjpos button x 25
setobjpos button y 20

objpushable box 1
objpushable button 0

// 3. YIELD UNTIL COMPLETED TO TRIGGER SUCCESS HUD TEXT EVENT
waituntiltouch box button
displaytext "You win!"
